A Bad Example


public class Sword {
    public void hit(String target) {
        Log.d(String.format("Sword chopped %s in half", target));
    }
}


public class Ninja {
    private final mSword;
    public Ninja() {
        mSword = new Sword;
    }

    public void attack(String target) {
        mSword.hit(target);
    }
}


/* Inside of a terrible Activity.onCreate method */
public void onCreate() {
    Ninja ninja = new Ninja();
    ninja.attack("zombie");
}

Output: Sword chopped zombie in half

  • As you probably expected chopped zombie in half, but what if we wanted to switch sword with something else?
  • This is a bad example of writing code. It violates the open close principle.
